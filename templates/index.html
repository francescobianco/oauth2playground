Get OAuth2 access tokens from your terminal — no web UI needed

How it works

  1. Copy the command below and paste it into your terminal
  2. Enter your OAuth2 credentials (client_id, secret, endpoints)
  3. Your browser opens — log in and authorize the app
  4. The terminal captures the redirect via netcat and exchanges the code
  5. Your access token appears right in the terminal

One-liner (interactive)

Copy and run this in your terminal:

curl -s {{.ServerURL}}/start | bashclick to copy

Requires curl, nc (netcat), and python3 on your machine.

▶ Google quick start

Google OAuth2 is pre-configured. Use two terminals:

Terminal 1 — start the request (connection stays open):

curl -v {{.ServerURL}}/api/google/tokenclick to copy

Terminal 2 — listen for the callback and send the code back:

RESP=$(printf 'HTTP/1.1 200 OK\r\n\r\nOK' | nc -l 8080) && CODE=$(echo "$RESP" | head -1 | sed 's/.*code=\([^& ]*\).*/\1/') && STATE=$(echo "$RESP" | head -1 | sed 's/.*state=\([^& ]*\).*/\1/') && curl -X POST {{.ServerURL}}/api/google/callback -d "code=$CODE&state=$STATE"click to copy

After authorizing in the browser, the redirect goes to netcat. The code is then sent back and Terminal 1 receives the token.

{{else}}

Google one-step

Set environment variables to enable the Google quick-start endpoint:

export OAUTH2PLAYGROUND_GOOGLE_CLIENT_ID=your_client_id
export OAUTH2PLAYGROUND_GOOGLE_CLIENT_SECRET=your_client_secret
export OAUTH2PLAYGROUND_GOOGLE_REDIRECT_URI=http://localhost:8080
export OAUTH2PLAYGROUND_GOOGLE_SCOPES="openid email profile"

Then restart the server.
